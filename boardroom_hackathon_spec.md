## 0. 一句话定位

**"ChatGPT 会同意你，董事会不会。"**

The Boardroom 是一人公司创始人的决策治理层：把一个待决策问题交给一个由数据驱动、程序性反谄媚的多 agent 董事会，经过盲评、交叉质证、结构化异议，产出一份带异议记录（dissent log）和否决线（kill criteria）的正式决策备忘录（Decision Memo）——建议权归董事会，**投票权永远只属于创始人**。

一句话给评委的升维："一人公司缺的从来不是执行力，是治理结构。真公司有董事会纪要，一人公司现在也有了。"

## 2. 痛点 → 方案 → Agent 角色

### 痛点
OPC 最被忽视的缺口不是执行力，是**判断力的孤独**：
- AI 消灭了人手，却保留甚至加重了决策重量——agent 升级上来的每个决定，创始人都要在没有同事压力测试的情况下拍板
- 行业调研信号（pitch 中可引用，注明为二手来源）：独立创业者高压力比例约 35%（有员工的老板约 26%），约三分之一想过彻底放弃
- 直接问 LLM 的问题：谄媚（顺着你说）、和稀泥（"一方面…另一方面…"）、无记录、无追责

### 方案
一个结构化的对抗性决策系统：
1. 创始人提交一个决策问题（如"订阅价从 $29 提到 $49？"）
2. 3–4 个董事 agent 在**信息隔离**状态下、基于**各自专属的数据管道**独立形成立场（盲评轮）
3. 全部证据摊桌，交叉质证轮（每个董事可修正立场）
4. 产出正式 **Decision Memo**：立场摘要、异议记录、各董事概率估计、否决线
5. （演示态）决议签发后监察 agent 持续盯否决线指标，触线自动"紧急召开董事会"

### AI Agent 的角色分工
| Agent | 数据管道（信息不对称的来源） | 职责 |
|---|---|---|
| CFO 董事 | 只接财务数据（CSV/表格：流水、订阅数据、成本） | 只能用数字说话 |
| 市场董事 | 只有行业报告摘要 / web search（现场可用预置的行业资料替代） | 竞品、市场趋势视角 |
| 客户董事 | 只看用户评论、客服记录（预置数据集） | 客户反应与流失风险 |
| 风险董事（魔鬼代言人） | 合同/条款/合规材料 + **制度性唱反调授权** | 绩效按"找出的最强反对理由"衡量，不按和气程度 |
| 主持 agent（Chair） | 全量可见，但只在辩论轮之后介入 | 主持流程、检测共识度、汇总备忘录 |
| 监察 agent（Sentinel，演示态） | 决议中的否决线指标 | 触线时重新召集董事会 |

## 3. 核心设计原则（这是护城河，实现时不可妥协的部分）

### 3.1 信息不对称 = 延迟融合，不是永久肢解
**关键澄清（评委可能挑战，自己也要想清楚）**：不对称管的不是信息总量，而是信息进入讨论的**顺序**。
- 阶段一：各董事隔离，只深挖自己的证据流，独立形成立场
- 阶段二：全部证据公开，交叉质证，每个董事看全量信息并可修正立场
- 最终决议在**信息完整**的前提下写成，没有任何信息被丢弃，被推迟的只是"融合"这个动作
- 理论根基：德尔菲法（先独立估计、再受控共享、再修正）+ 盲审逻辑

为什么不直接让一个模型看全量数据（Q&A 预案，也是设计依据）：
1. **注意力稀释**：长上下文中段信息利用率下降；强制单一证据流被榨干（类比：辩护律师的价值不在于无知于控方证据，而在于其职责保证辩方证据被最强力开采）
2. **错误相关**：共享上下文导致锚定，"五个视角"退化为一个视角的五次复读；集成有效性的前提是成员错误互不相关
3. **谄媚**：创始人倾向写在共享上下文里所有顾问都看得见；盲评阶段物理上看不到，就谄媚不起来

### 3.2 程序性反谄媚（做成看得见的产品特性）
- **盲评第一轮**：创始人的个人倾向字段对董事隐藏；董事在互相隔离的上下文中并行产出开场立场
- **消灭不掉的反方**：风险董事被制度性指派唱反调
- **共识警报**：第一轮分歧度过低 → 主持 agent 不庆祝，触发强制追问轮；UI 上显示"分歧度过低，强制深挖"提示（反直觉设计，展示决策科学素养）

### 3.3 输出是治理文件，不是聊天记录
Decision Memo 固定结构：
```
决策问题 | 各董事立场摘要 | 异议记录（少数派意见原文保留，不许被综合掉）
各董事概率估计（逼 agent 给数字不给态度） | 否决线 kill criteria
（如"6 周内流失率 > 5% 则本决议自动作废，重开董事会"） | 创始人最终裁决（永远留白给人）
```

### 3.4 董事会不散会（演示态即可）
- 监察 agent 盯否决线 → 触线推送"紧急董事会已召集，客户董事的原始异议被验证"
- **董事信誉分**：预置几条历史决策记录展示机制（谁的概率估计接近事实、权重上调；连续看走眼可"解雇"）。pitch 时坦白这是路线图功能，坦白本身加分
- 回应"这和直接问一个强模型有什么区别"：单模型没有可核查的判断记录；每个董事有 track record，板凳随公司一起变准

### 3.5 事前验尸（Premortem）轮
辩论后、签发前：指定一位董事想象"12 个月后这个决定失败了，验尸报告怎么写"，输出写入备忘录风险栏。

## 4. 技术架构建议

- **编排**：LangGraph（用户熟练）。图结构：
  `intake → [4 个董事节点并行，隔离 state] → 共识度检测 → (低分歧 → 强制追问轮) → 证据全公开 → 交叉质证轮(1–2 round) → premortem 节点 → memo 生成 → (演示态) sentinel`
- **状态设计**：每个董事节点独立的 context/state 分区是"隔离"的技术落点；全公开阶段才 merge
- **模型**：Anthropic API（比赛语境加分）；董事可用同一模型不同隔离上下文——差异化来自数据管道而非模型多样性（这个立场要能在 Q&A 里讲清楚）
- **数据接入**：demo 用**预置本地数据集**（CSV 财务表 + 评论 JSON + 条款 md），架构上留 MCP connector 的接口位并在 pitch 提到"生产环境经 MCP 连 Stripe/表格/店铺后台"
- **前端**：从简。Streamlit 或单页 Next.js/静态页；核心界面 = ①董事会进行中的可视化（各董事立场卡片 + 分歧度仪表）②Decision Memo 渲染页。**不做**花哨动画
- **demo 方差控制（重要）**：核心演示场景全程**预跑并缓存**每一轮的输出，现场默认播放缓存流（逐字流式渲染以保真实感），保留"live 重跑"按钮作为备份。只接受评委从预置的 2–3 个场景中挑选，不接受任意话题即兴

## 5. Demo 场景与剧本

**主场景**：SaaS 订阅涨价决策（$29 → $49），预置真实感数据：
- 财务 CSV：月流水、订阅曲线、CAC、毛利
- 评论 JSON：约 30 条用户评论（部分价格敏感）
- 条款 md：现有用户价格承诺条款（给风险董事埋的钩子：老用户涨价可能违反承诺 → 这是"幸存分歧"的引爆点）

**剧本节拍**（3 分钟 pitch 内嵌 demo 时的顺序）：
1. **自指开场（30 秒）**："赛前我在两个参赛方向之间纠结了三天，我把它们喂给了这个董事会的原型——它投票选了它自己。这是当时的异议记录。"（需要赛前真的用原型跑一次并留存截图）
2. 提交决策问题 → 盲评轮：四张立场卡片并行浮现，CFO 说涨（数字），客户董事说炸（评论）
3. 共识警报或分歧展示 → 交叉质证：客户董事看到财务数据后修正立场，风险董事的条款异议**幸存**
4. Premortem 一屏
5. Decision Memo 渲染：异议记录 + 概率 + 否决线 + 创始人裁决留白 → 现场点击"创始人裁决"
6. 结尾钩子：手机通知截图 "⚠️ 流失率触及 5.2%，紧急董事会已召集"
7. （若时间允许）**对照画面**：同一问题丢给裸单模型的"一方面另一方面"截图 vs 董事会的尖锐立场+概率+否决线，并排放，不解说

**Taglines 备选**：
- "ChatGPT 会同意你，董事会不会"
- "我不是在制造分歧，我是在阻止过早的一致——分歧免费涌现，被验证的一致才是产品"

## 8. 评委 Q&A 预案

**Q: 这和直接问一个强模型有什么区别？**
A: 三个机制差异：盲评防锚定防谄媚（程序保证，不靠 prompt 恳求）；专属证据管道保证每条证据流被榨干而非在长上下文里被稀释；输出是有异议记录、概率、否决线的可追责治理文件，且每个董事积累可核查的 track record。

**Q: 故意让它们信息不对称，不是故意制造吵架吗？**
A: 不对称管的是信息进入讨论的顺序，不是总量——两阶段设计，最终决议基于全量信息。吵架不是目标，是诊断信号：全公开后自行消失的分歧是信息差，顺手修复；幸存的分歧才是价值观/风险偏好层面的真分歧，这正是需要创始人裁决的东西。单模型直接交付一份"已经和好了的综合"，给不了这个分类。

**Q: 多 agent 辩论真的比单模型准吗？**
A: 诚实回答：学术证据是混合的——有研究显示辩论提升事实性与推理，也有复现发现增益不稳定。所以我们不承诺"必然更准"，承诺的是机制层面扎实的三件事：防锚定、防谄媚、保证证据开采深度。（这个诚实本身是立场。）

**Q: 董事用的都是同一个模型，多样性从哪来？**
A: 多样性的来源是数据管道和职责授权，不是模型品牌——真实董事会的分歧也不来自董事们大脑构造不同，而来自各自掌握的事实和承担的职责不同。架构上支持异构模型，但那是优化项不是前提。

**Q: 生产环境数据怎么接？**
A: MCP。CFO 连 Stripe/表格，客户董事连评论与客服系统，接口位已留。

**Q: 商业模式？**
A: 按董事会席位/决策次数订阅；高价值场景（定价、融资条款、大额采购）单次付费也成立。目标用户就是本次黑客松定义的人群：正在指挥 agent 舰队、却没有任何人帮他压力测试判断的 OPC 创始人。

## 10. 诚实边界（写给自己，也备评委深挖时交出）

- 治理"判断质量"不等于保证"决策正确"——系统提升的是决策程序质量，不是预言能力
- 信誉分在 7 小时内只能是演示态，真实校准需要时间——坦白为路线图功能
- "多 agent 辩论优于单模型"的学术证据混合——只承诺机制，不承诺准确率
- 初筛机制与评分权重系推断（主办方未公布）——所有针对评委的优化都是有依据的赌注，不是事实
