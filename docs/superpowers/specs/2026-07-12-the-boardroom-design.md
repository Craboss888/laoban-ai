## 2 · 技术架构(三个独立单元,经一个 schema 通信)

```
数据包 data/ ──▶ 生成器 scripts/generate.py ──▶ 缓存 output/cache.js ──▶ 查看页 site/index.html
```

- **无 LangGraph**:创始人不熟;demo 走缓存不需要运行时框架;信息隔离用普通 Python 函数边界表达("董事函数只接收自己的数据切片")反而更好解释。对外口径仍是 orchestration graph(概念如此)。
- **无 API key**:生成器写成可真调 Anthropic API 的完整形态(架构真实、评委可读),demo 缓存内容由 Claude(Opus)按各董事数据切片撰写,schema 与生成器输出严格一致。live 重跑不做。
- **查看页零外部依赖**(无 CDN、无网络请求),缓存 inline;`file://` 直开,现场不可能崩。

## 3 · 角色

- **CEO = 用户本人**:坐正中,裁决权唯一归属;拥有"结束讨论"权。
- **默认董事 ×4**(每人只见自己的数据管道):
  | 董事 | 专属数据 | 职责 |
  |---|---|---|
  | CFO | finance.csv | 只用数字说话 |
  | 客户董事 | reviews.json | 客户反应与流失风险 |
  | Mr. Opposite(风险/魔鬼代言人) | terms.md | **制度性唱反调授权**,绩效按最强反对理由计 |
  | 市场董事 | market.md | 竞品与趋势 |
- **秘书(兼主持,不投票)**:格式化议题、主持轮次、共识检测、起草 memo。
- 选角界面另列**领域董事**(法务/供应链/公关/电商 PM)标 coming soon,注"由领域专有语料 + 方法论构建"(路线图口径)。**不做名人董事**(与"非 prompt 人设"主张矛盾 + 形象权风险,已裁决)。

## 4 · 六幕 user journey(demo 实呈,缓存回放)

0. **组建董事会**:选角(默认 4 董事 + 秘书;领域董事 coming soon)+「已连接数据源」卡片(mock;生产态 = MCP 连 Stripe/客服系统,pitch 一句话)
1. **提交议题**:口述大白话(预填)→ 秘书当场整理成结构化议题卡(决策问题/背景/期限);创始人个人倾向单独录入并标注"**盲评期间对董事隐藏**"
2. **盲评轮(墙起)**:四信息舱并行打字,各自只见自己数据;产出四张立场卡(立场 + 概率 + 关键证据);分歧度仪表启动
3. **墙落 · 自由讨论**:舱壁落下、证据全公开摊上圆桌;多方对话流(真开会你来我往),每条发言挂「发言人 + 当前立场 chip + 引用证据标签」;两大节拍自动高亮:①客户董事见财务数据后**当众修正立场**(立场值滑动 + diff 动画)②Mr. Opposite 条款异议被多方挑战**仍幸存**(→ 进异议记录)
4. **休会**:CEO 点「结束讨论,请秘书起草备忘录」(结束权在创始人)
5. **Decision Memo + 裁决**:立场摘要 / **异议记录(少数派原文保留)** / 各董事概率估计 / **否决线**(如"6 周内流失率 >5% 则本决议自动作废,重开董事会")/ premortem 风险节 / **创始人裁决块**——现场点击裁决。**提交首图取自本幕。**

## 5 · 数据包(全 mock,自由撰写)

- `finance.csv`:月流水/订阅曲线/CAC/毛利——数字支持涨价 → 仅 CFO
- `reviews.json`:~30 条评论,部分价格敏感 → 仅客户董事
- `terms.md`:**老用户价格承诺条款 = 幸存分歧引爆点** → 仅 Mr. Opposite
- `market.md`:竞品定价/趋势 → 仅市场董事
- `case.json`:议题原述 + 结构化议题 + 创始人隐藏倾向

## 6 · UI/UX 方向

- **活力简约**;会议室空间隐喻:CEO 居中、董事环绕圆桌;**墙起 → 墙落**是核心转场,它本身就是机制的可视化(不是装饰动效)。
- 席位 = 抽象档案卡 + **数据来源徽章**("仅财务数据"等),不做拟真人物/卡通形象。
- 动效克制而有目的:并行打字、仪表跳动、立场 diff、墙落、收束;不炫技。
- 幕 5 memo = 严肃治理文件排版,与前幕活力形成**收束反差**(七嘴八舌 → 一纸权威文件)。
- 默认浅色活力主题,做到一种主题完美即可;对着真像素再调。

## 7 · 缓存 schema(概要)

```
scenario {
  meta{ title, tagline, question_raw, question_structured, hidden_bias },
  roles[ {id, name, badge, dataset, color} ],
  datasets[ {id, filename, preview_rows} ],
  acts[
    act2_blind:   [ {role, stance, prob, evidence[], text_stream} ],
    act3_debate:  [ {speaker, text, stance_chip, refs[], highlight?:"revision"|"dissent_survives"} ],
    act4_adjourn: { trigger:"ceo" },
  ],
  memo{ positions[], dissent[], probabilities[], kill_criteria, premortem, ruling:null }
}
```

## 8 · 范围(demo version 纪律)

## 9 · pitch 口径(诚实边界)

- **今天可证**:同一模型 + 物理隔离的数据管道 → 分歧确定来自**信息**而非人设(变量锁死,比"不同模型"更锋利)。
- **路线图(明说是路线图)**:领域语料强化(法务读判例、PM 读方法论)→ 异构模型适配 → **董事 marketplace**("最懂电商的产品董事"可订阅)。
- 商业模式:按席位/决策次数订阅 + 领域董事 marketplace。
- 绝不把路线图说成现状——"诚实本身是立场"(上位 spec §10)。
